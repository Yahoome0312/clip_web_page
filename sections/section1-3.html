1.3 The Core Idea of CLIP

CLIP's core insight was simple but profound: Instead of predicting exact words (Generative) or classifying into fixed buckets (Supervised), it is much easier to simply determine:

"Match or No Match?"

To implement this, CLIP uses two separate neural networks:

  • Image Encoder: Processes images into a vector $I_f$.
  • Text Encoder: Processes text into a vector $T_f$.

These embeddings are mapped into a Shared Latent Space. If an image and a piece of text are related, their embedding vectors should be close to each other (high cosine similarity). If unrelated, they should be far apart.

The key challenge is how to train the model so that this property emerges naturally. CLIP solves this using a Contrastive Learning objective, which we will explore in the next chapter.
